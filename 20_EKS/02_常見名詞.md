# 常見名詞

_EKS 領域中常見的術語或專業名詞_

<br>

1. SRE（Site Reliability Engineering） 是一種方法論，通過應用軟件工程技術來提高 IT 系統的可靠性和效率，它的核心理念包括服務可靠性、SLO 和 SLA、錯誤預算、自動化和工程化運維。SRE 將運維視為一個工程問題，通過編寫代碼和使用自動化工具來解決運維挑戰。

<br>

2. SLO（Service Level Objective）：服務級別目標，是對服務性能和可靠性的具體期望值。

<br>

3. SLA（Service Level Agreement）：服務級別協議，是與客戶或用戶之間的服務性能和可靠性的正式協議。

<br>

4. DevOps 是一種方法論，通過促進開發和運營團隊之間的協作和自動化，來提高軟件開發和交付的效率和質量。它的核心是協作、自動化、持續集成和持續交付，以及監控和反饋。是一種結合軟件開發（Development）和 IT 運維（Operations）的方法論和文化，旨在通過協作和自動化來提高軟件開發和交付的速度和質量。常見的 DevOps 工具包括版本控制類 Git、GitHub、GitLab；CI/CD 類 Jenkins、Travis CI、CircleCI、AWS CodePipeline；配置管理類 Ansible、Chef、Puppet；容器化類 Docker、Kubernetes；監控類 Prometheus、Grafana、Nagios。

<br>

5. CI/CD 持續集成和持續交付，持續集成是指頻繁地將代碼更改集成到主代碼庫中，並自動化構建和測試。持續交付是指自動將構建的應用部署到生產環境或準生產環境中，確保軟件隨時可部署。常見的有 Jenkins、Travis CI、CircleCI、AWS CodePipeline。
